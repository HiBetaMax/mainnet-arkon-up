Vercel deploy notes

1. Set Root Directory to arkon-runtime if the repo contains this folder.
2. Add env var ENABLE_EXPERIMENTAL_COREPACK=1 in Vercel project settings.
3. Redeploy.

This package was updated to pin:
- Node: >=22.12.0 <23
- pnpm: 10.25.0

Note:
A pnpm-lock.yaml is not included in this zip because the packaging environment here cannot reach the npm registry to generate one.
The included vercel.json forces pnpm install/build, and package.json declares packageManager for Corepack.
If you want maximum determinism, run locally once inside arkon-runtime:
  corepack enable
  corepack prepare pnpm@10.25.0 --activate
  pnpm install
Then commit the generated pnpm-lock.yaml.
